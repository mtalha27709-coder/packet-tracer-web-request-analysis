# 🌐 Packet Tracer - Web Request Analysis



## 📌 Overview


This project demonstrates how a client communicates with a web server over a network using HTTP. The lab was performed using Cisco Packet Tracer to observe real-time traffic flow, DNS resolution, and TCP-based communication.



---



## 🎯 Objectives


- Understand client-server communication
- Observe DNS resolution (domain → IP)
- Analyze HTTP requests and responses
- Study TCP traffic and packet flow



---



## 🛠️ Tools Used


- Cisco Packet Tracer



---



## 🔍 Lab Tasks



### ✅ Part 1: Verify Connectivity


- Used `ping ciscolearn.web.com`
- Observed DNS resolving domain to IP address
- Verified successful communication using ICMP



---



### 🌐 Part 2: Access Web Server
- Opened Web Browser in Packet Tracer
- Accessed the website using domain name
- Observed how HTTP works in real-time



---



### 💻 Part 3: View HTML Code
- Accessed server services (HTTP)
- Viewed and analyzed `index.html`
- Compared server-side code with browser output

  

---


### 📡 Part 4: Traffic Analysis (Simulation Mode)


- Switched to Simulation Mode
- Applied filters (TCP, HTTP)
- Created Complex PDU (HTTP request)
- Observed:
  - TCP handshake
  - HTTP request/response
  - Packet flow between client and server
  - Increased traffic due to TCP acknowledgments



---



## 💡 Key Learnings


- DNS converts domain names into IP addresses
- HTTP works over TCP, requiring multiple packets
- TCP ensures reliable communication (handshake & acknowledgments)
- Real-world traffic includes overhead, not just simple requests



---



## 🧠 Skills Gained


- Network Fundamentals (TCP/IP, DNS, HTTP)
- Packet Analysis & Traffic Observation
- Basic Troubleshooting
- Client-Server Communication
- Hands-on with Cisco Packet Tracer

This lab provided a practical understanding of how web requests travel across a network. It strengthened my ability to analyze network behavior and understand real-world communication processes.
